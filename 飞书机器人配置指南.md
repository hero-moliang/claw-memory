## 第一步：创建飞书应用（在飞书开放平台）

### 1. 访问开放平台
- 国内版：https://open.feishu.cn/app
- 国际版(Lark)：https://open.larksuite.com/app （需在配置中设置 `domain: "lark"`）

### 2. 创建企业自建应用
点击 **创建企业自建应用** → 填写名称和描述 → 选择图标

### 3. 获取应用凭证
进入 **凭证与基础信息** 页面，复制：
- **App ID**（格式：`cli_xxx`）
- **App Secret**（⚠️ 妥善保管，不要泄露）

### 4. 配置权限（关键！）
进入 **权限管理** → 点击 **批量导入** → 粘贴以下 JSON：

```json
{
  "scopes": {
    "tenant": [
      "aily:file:read",
      "aily:file:write",
      "application:application.app_message_stats.overview:readonly",
      "application:application:self_manage",
      "application:bot.menu:write",
      "cardkit:card:write",
      "contact:user.employee_id:readonly",
      "corehr:file:download",
      "docs:document.content:read",
      "event:ip_list",
      "im:chat",
      "im:chat.access_event.bot_p2p_chat:read",
      "im:chat.members:bot_access",
      "im:message",
      "im:message.group_at_msg:readonly",
      "im:message.group_msg",
      "im:message.p2p_msg:readonly",
      "im:message:readonly",
      "im:message:send_as_bot",
      "im:resource",
      "sheets:spreadsheet",
      "wiki:wiki:readonly"
    ],
    "user": ["aily:file:read", "aily:file:write", "im:chat.access_event.bot_p2p_chat:read"]
  }
}
```

### 5. 启用机器人能力
进入 **应用能力** → **机器人** → 开启机器人 → 设置机器人名称

### 6. 配置事件订阅（⚠️ 注意顺序）
**先确保：**
1. 已运行 `openclaw channels add` 添加了 Feishu 渠道
2. 网关已启动（`openclaw gateway status` 检查）

然后：
- 进入 **事件订阅**
- 选择 **使用长连接接收事件**（WebSocket 模式）
- 添加事件：`im.message.receive_v1`

### 7. 发布应用
进入 **版本管理与发布** → 创建版本 → 提交审核 → 等待审批

## 第二步：配置 OpenClaw

### 方式一：命令行配置（推荐）

```bash
openclaw channels add
```
- 选择 **Feishu**
- 输入 App ID 和 App Secret
- 完成！

### 方式二：直接编辑配置文件

编辑 `~/.openclaw/openclaw.json`：

```json5
{
  channels: {
    feishu: {
      enabled: true,
      dmPolicy: "pairing",  // 私聊策略
      accounts: {
        main: {
          appId: "cli_xxx",      // 你的 App ID
          appSecret: "xxx",      // 你的 App Secret
          botName: "我的AI助手",  // 机器人名字
        },
      },
    },
  },
}
```

## 第三步：启动并测试

### 1. 启动网关
```bash
openclaw gateway
# 或后台运行
openclaw gateway start
```

### 2. 检查状态
```bash
openclaw gateway status
openclaw logs --follow
```

### 3. 测试私聊
- 在飞书中搜索你的机器人
- 发送一条消息
- 机器人会回复一个**配对码**

### 4. 批准配对
```bash
# 查看待批准列表
openclaw pairing list feishu

# 批准配对
openclaw pairing approve feishu <配对码>
```

批准后就可以正常对话了！

## 🔧 常用配置选项

### 私聊策略 (`dmPolicy`)
| 值 | 说明 |
|---|-----|
| `"pairing"` | 默认，新用户需配对码批准 |
| `"allowlist"` | 仅白名单用户可访问 |
| `"open"` | 允许所有人（需配合 `allowFrom: ["*"]`）|
| `"disabled"` | 禁止私聊 |

### 群组配置
```json5
{
  channels: {
    feishu: {
      groupPolicy: "open",      // "open"|"allowlist"|"disabled"
      groupAllowFrom: ["oc_xxx"], // 允许的群组ID列表
      groups: {
        "oc_xxx": {
          requireMention: true,   // 是否需要@机器人
          enabled: true,
        },
      },
    },
  },
}
```

### 获取 ID 的方法
启动网关后，查看日志：
```bash
openclaw logs --follow
```
- 用户 Open ID：`ou_xxx`
- 群组 ID：`oc_xxx`

## ⚠️ 常见问题

### 1. 机器人收不到消息
- [ ] 应用是否已发布并审批通过？
- [ ] 事件订阅是否选择了 **长连接** 模式？
- [ ] 权限是否全部添加？
- [ ] 网关是否正在运行？

### 2. 机器人在群组中不响应
- [ ] 机器人是否已添加到群组？
- [ ] 是否 @了机器人（默认需要）？
- [ ] `groupPolicy` 是否为 `"disabled"`？

### 3. 发送消息失败
- [ ] 是否有 `im:message:send_as_bot` 权限？
- [ ] 应用是否已发布？

### 4. App Secret 泄露
1. 在飞书开放平台重置 App Secret
2. 更新配置文件
3. 重启网关

## 🛠️ 高级功能

### 工具开关（在 `~/.openclaw/openclaw.json` 中）
```json5
{
  channels: {
    feishu: {
      tools: {
        doc: true,    // 文档读写（默认开启）

      },
    },
  },
}
```

### 流式输出
```json5
{
  channels: {
    feishu: {
      streaming: true,      // 启用流式卡片（默认开启）
      blockStreaming: true, // 块级流式（默认开启）
    },
  },
}
```

### 多账号配置
```json5
{
  channels: {
    feishu: {
      defaultAccount: "main",
      accounts: {
        main: {
          appId: "cli_xxx",

          botName: "主机器人",
        },
        backup: {
          appId: "cli_yyy",
          appSecret: "yyy",
          botName: "备用机器人",
          enabled: false,
        },
      },
    },
  },
}
```

## 💡 小布丁的 Tips

1. **一定要先发布应用再测试** —— 未发布的应用无法接收消息
2. **事件订阅选长连接** —— 不需要公网 URL，最简单
3. **记得批准配对码** —— 否则私聊无法使用
4. **群组需要 @机器人** —— 默认配置下需要 @ 才能触发
5. **Lark 国际版要设置 domain** —— 配置 `"domain": "lark"`
